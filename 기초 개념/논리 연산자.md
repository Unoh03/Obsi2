> [!danger] 🚨 네트워크 엔지니어가 논리 연산을 알아야 하는 이유
> 라우터는 십진수(`192.168.1.1`)를 모른다. 오직 32비트 이진수의 **AND, OR, NOT 연산**을 통해 네트워크 ID를 추출하고, 와일드카드 마스크를 계산하며, ACL 방화벽 규칙을 적용한다.

> [!info]- 정보 출처 및 생산일자
> - **출처 1:** 조지 불(George Boole)의 "논리의 수학적 분석 (The Mathematical Analysis of Logic)" - 불 대수(Boolean Algebra)의 기원.
> - **출처 2:** 클로드 섀넌(Claude Shannon)의 MIT 석사 논문 - 릴레이 스위치를 이용한 논리 연산 구현.
> - **생산일자:** 1847년(이론 정립) ~ 1937년(디지털 회로 적용)

# 1. 🟢 AND (논리곱) : "둘 다 1이어야 1이다"
- **기호:** `&` (비트 연산), `&&` (논리 연산)
- **원리:** 조건이 **모두 참(True)**일 때만 참을 반환하는 깐깐한 문지기.
- **네트워크 실무 적용 (Subnetting):** 라우터가 IP 주소와 서브넷 마스크를 **AND 연산**하여 동네 이름(Network ID)을 알아낼 때 쓴다.
- **네트워크 실무 적용 (Route Summarization):** 여러 개의 라우팅 경로를 하나로 묶어버리는 '경로 요약(Supernetting)'을 계산할 때 비트 단위 OR 연산의 개념이 들어간다.
# 2. 🔵 OR (논리합) : "하나라도 1이면 1이다"
- **기호:** `|` (비트 연산), `||` (논리 연산)
- **원리: ** 조건 중 **하나라도 참(True)**이면 참을 반환하는 관대한 문지기.
- **네트워크 실무 적용 (QoS & ACL):** 방화벽(ACL)에서 "출발지가 A **또는(OR)** B일 때 허용하라"는 다중 조건을 걸거나, 특정 비트 플래그를 강제로 1로 세팅(Masking)할 때 사용한다.
# 3. 🔴 NOT (논리부정) : "청개구리 (0은 1로, 1은 0으로)"
- **기호:** `~` (비트 반전), `!` (논리 부정)
- **원리:** 입력된 값을 **반대로 뒤집어버린다.**
- **네트워크 실무 적용 (Wildcard Mask):** OSPF나 ACL에서 쓰는 와일드카드 마스크는 서브넷 마스크를 **NOT 연산(비트 반전)**한 결과물이다.
  - 서브넷 마스크: `255.255.255.0` (`11111111.11111111.11111111.00000000`)
  - NOT 연산 결과: `0.0.0.255` (`00000000.00000000.00000000.11111111`)
# 4. 🟣 XOR (배타적 논리합) : "서로 달라야 1이다"
- **기호:** `^`
- **원리:** 두 값이 **서로 다를 때만 참(True)**을 반환한다. (같으면 0)
- **네트워크 실무 적용 (Cryptography & Checksum):** VPN(IPsec)에서 데이터를 암호화하거나, 패킷이 깨졌는지 확인하는 오류 검출(FCS, Checksum) 알고리즘의 핵심 뼈대다.

**[2] Python에서의 논리/비트 연산자**
```python
# Python 비트 연산자 예시
a = 12  # 이진수: 1100
b = 10  # 이진수: 1010

print(a & b)  # AND 연산 -> 1000 (8)
print(a | b)  # OR 연산  -> 1110 (14)
print(a ^ b)  # XOR 연산 -> 0110 (6)
print(~a)     # NOT 연산 -> 0011 (비트 반전, 부호 비트 포함 시 -13)
```

**[3] Java/C 계열에서의 논리 연산자 (조건문)**
```java
// Java 조건문에서의 논리 연산
boolean isPortUp = true;
boolean isProtocolUp = false;

// AND (&&): 둘 다 true여야 통과 (라우터 인터페이스 상태 확인 시)
if (isPortUp && isProtocolUp) {
    System.out.println("Interface is UP/UP");
} else {
    System.out.println("Interface is DOWN"); // 이 코드가 실행됨
}

// NOT (!): 상태 뒤집기
if (!isProtocolUp) {
    System.out.println("Protocol is down, check encapsulation!");
}
