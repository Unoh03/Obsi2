## 💀 [RCA] 데이터의 '무(無)': 4가지 상태의 물리적 차이

> [!danger] 🚨 아키텍트의 팩트 폭격: "0은 숫자고, 공백은 문자열이다. null은 '의도된 빈칸'이고, undefined는 '실수로 만든 빈칸'이다. 이걸 섞어 쓰는 순간 시스템은 런타임 에러로 폭발한다."

### 🔍 1. 비유를 통한 아키텍처 이해 (The Box Analogy)

| 상태 | 비유 (상자) | 데이터 타입 | 설명 |
| :--- | :--- | :--- | :--- |
| **`0`** | 상자 안에 숫자 '0'이 들어있음 | **Number** | 값(Value)이 존재함. 단지 그 값이 0일 뿐. |
| **`""` (공백)** | 상자 안에 빈 종이가 들어있음 | **String** | 문자열 객체는 생성됨. 단지 내용(Length)이 0일 뿐. |
| **`null`** | 상자가 비어있다고 명시적으로 표시함 | **Object** | "여기는 값이 없다"라고 개발자가 **의도적으로** 비워둔 상태. |
| **`undefined`** | 상자 자체가 어디 있는지 모름 | **Undefined** | 변수를 선언만 하고 값을 넣지 않음. **실수나 정의되지 않음**을 의미. |

---

### 🚨 2. 자바스크립트의 멍청한 비교 (Falsy Values)
JS의 `if`문에서는 이 4가지가 전부 `false`로 취급된다. 그래서 **`===` (엄격한 비교)**를 쓰지 않으면 대참사가 난다.

```javascript
0 == false;         // true (미친 짓)
"" == false;        // true (미친 짓 2)
null == undefined;  // true (미친 짓 3)

0 === false;        // false (정상)
null === undefined; // false (정상)
