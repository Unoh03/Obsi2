### 1. VI/VIM 모드 종류 및 전환
>[!warning] 🚨 vi의 핵심: 현재 내가 무슨 모드인지 파악하는 것이 생명이다. 길을 잃으면 무조건 `ESC`를 연타하라.

| 모드 | 설명 | 전환 방법 |
| :--- | :--- | :--- |
| **일반 모드 (Normal Mode)** | vi를 실행하면 최초로 진입하는 기본 모드 | `ESC` |
| **입력 모드 (Insert Mode)** | 텍스트를 직접 타이핑하여 입력하는 모드 | `a`, `i`, `o` |
| **비주얼 모드 (Visual Mode)** | 텍스트를 블록 지정(선택)하는 모드 | `v`, `V`, `Ctrl+v` |
| **명령 행 모드 (Command Line)** | 저장, 종료, 환경 설정 등을 입력하는 모드 | `:` |
| **검색 (Search)** | 파일 내에서 특정 문자열을 찾는 모드 | `/` 또는 `?` |

---

### 2. 커서 이동 (Navigation)
> [!tip] 💡 고인물은 방향키(화살표)를 쓰지 않는다. 오직 알파벳만으로 커서를 날아다니게 하라.

| 단축키 | 설명 |
| :--- | :--- |
| `gg` | 파일의 **첫 번째 줄**로 이동 |
| `G` | 파일의 **마지막 줄**로 이동 |
| `nG` 또는 `ngg` | `n`번째 줄로 이동 (예: `22G` ➡️ 22번째 줄로 이동) |
| `Ctrl+u` / `Ctrl+d` | 화면을 위(Up) / 아래(Down)로 반 페이지씩 스크롤 |
| `w` | 다음 단어의 **처음**으로 이동 (Word) |
| `e` | 현재 단어의 **끝**으로 이동 (End) |
| `f<알파벳>` | 현재 행에서 입력한 특정 알파벳 위치로 점프 (Find) |
| `^` | 현재 행의 **맨 앞**으로 이동 (공백 제외) |
| `$` | 현재 행의 **맨 끝**으로 이동 |

---

### 3. 텍스트 조작 (삭제/복사/붙여넣기)
> [!danger] 🚨 `dd`는 단순한 삭제가 아니라 '잘라내기(Cut)'다. 지운 내용은 클립보드에 남아 `p`로 붙여넣을 수 있다.

| 단축키 | 설명 |
| :--- | :--- |
| `dd` | 현재 행 전체 삭제 (잘라내기) |
| `D` | 커서 위치부터 그 행의 끝까지 삭제 |
| `yy` | 현재 행 전체 복사 (Yank) |
| `p` / `P` | 복사하거나 잘라낸 내용을 커서 뒤(`p`) 또는 앞(`P`)에 붙여넣기 (Paste) |
| `u` | 실행 취소, 이전 작업으로 되돌리기 (Undo) |
| `Ctrl+r` | 다시 실행, Undo 한 것을 되돌리기 (Redo) |

---

### 4. 입력 및 비주얼 모드

#### 입력 모드 (Insert Mode) 진입
| 단축키 | 설명 |
| :--- | :--- |
| `a` | 현재 커서 **뒤(오른쪽)**에서 입력 모드로 전환 (Append) |
| `i` | 현재 커서 **앞(왼쪽)**에서 입력 모드로 전환 (Insert) |
| `o` | 현재 행의 **아래에 새 행을 추가**하고 입력 모드로 전환 (Open) |

#### 비주얼 모드 (Visual Mode) 블록 지정
| 단축키 | 설명 |
| :--- | :--- |
| `v` | **문자 단위**로 블록 선택 |
| `V` | **행(줄) 단위**로 블록 선택 |
| `Ctrl+v` | 커서를 기준으로 **사각형(블록) 형태**로 선택 (주석 일괄 처리 시 유용) |
| `y` | 선택 영역 복사 (Yank) |
| `d` | 선택 영역 삭제 (Delete) |

---

### 5. 명령 행 모드 (Command Line)
> [!warning] 🚨 무지성 `:wq`는 파일의 수정 시간(mtime)을 갱신하여 서버 오작동을 유발할 수 있다. 실무에서는 `:x`를 애용하라.

| 단축키 | 설명 |
| :--- | :--- |
| `:w` | 파일 저장 (Write) |
| `:q` | 저장하지 않고 종료 (Quit) |
| `:wq` 또는 `:x` | 저장하고 종료 (`:x`는 변경 사항이 있을 때만 저장하는 스마트 종료) |
| `:q!` | 변경 사항을 무시하고 **강제 종료** (설정 파일 건드리다 꼬였을 때 비상 탈출구) |
| `:set number` | 에디터 왼쪽에 줄 번호 출력 |
| `:set nonumber` | 줄 번호 숨김 |
| `:set hlsearch` | 검색 결과에 하이라이트(형광펜) 표시 |
| `:set ignorecase` | 대/소문자 구분 없이 검색 |
| `:s/pattern/replacement/` | 검색한 문자열(`pattern`)을 새로운 문자열(`replacement`)로 교체 |

---

### 6. 검색 (Search)

| 단축키 | 설명 |
| :--- | :--- |
| `/` 또는 `?` | 커서를 기준으로 아래로(`/`) 또는 위로(`?`) 문자열 검색 |
| `n` | **다음** 검색 결과로 이동 (Next) |
| `N` | **이전** 검색 결과로 이동 (역방향) |



- 쿠버네티스에선 최대한 경량화 하다보니 vi를 사용.
- 비주얼 모드 몰라도 됨
- 1. **`:x` 또는 `ZZ` (스마트 저장 및 종료) - 🌟 실무 표준** - **원리:** 파일 내용이 변경된 경우에만 저장(Write)하고 종료. 변경되지 않았다면 그냥 종료(Quit). - **효과:** 불필요한 디스크 I/O 방지 및 파일 메타데이터(mtime) 보존.
  2. **`:q!` (강제 종료) - 🚨 비상 탈출구** - **원리:** 변경 사항을 무시하고 강제로 에디터를 종료. - **상황:** 핵심 설정 파일(`/etc/passwd` 등)을 열람하다가 실수로 오타를 냈을 때, 시스템 붕괴를 막기 위해 변경 사항을 폐기하고 도망칠 때 사용. 
  3. **`:w` (저장만 하기)** - 에디터를 끄지 않고 중간 저장만 할 때 사용. (긴 스크립트 작성 시 필수)
>[!danger]- 🚨 `:wq` 무지성 사용의 위험성 (mtime 갱신)
> `:wq`는 변경 사항 유무와 관계없이 무조건 디스크에 Write(I/O)를 발생시켜 파일의 **수정 시간(mtime)**을 갱신한다. 이는 파일 변경을 감지하여 작동하는 자동화 스크립트(CI/CD, Make 등)를 오작동하게 만드는 주범이다.

- `:!` cisco에서 do와 비슷함
  > [!tip] 🚨 `:` 와 `:!` 의 본질적 차이
> - **`:` (Internal Command):** `vi` 에디터 자체의 내장 명령어를 실행한다. (저장, 종료, 치환 등)
> - **`:!` (External Shell Command):** 에디터를 종료하지 않고, 호스트 OS의 쉘(Bash 등)을 호출하여 외부 리눅스 명령어를 실행하는 **'Shell Escape'** 기능이다.

- **🛠️ 실무 엔지니어의 `:!` 활용법 (생산성 극대화)**
  에디터 작업 중 흐름을 끊지 않고 OS와 상호작용할 때 사용한다.
  1. **`:!ls -l`** : 현재 디렉토리의 파일 목록 확인 후 복귀. 
  2. **`:!ping 8.8.8.8`** : 네트워크 상태 확인 후 복귀.
  3. **`:r !date`** : (응용) 리눅스의 `date` 명령어 실행 결과x를 **현재 커서 위치의 텍스트로 그대로 삽입(Read)**한다. (스크립트 작성 시 개꿀팁)

> [! danger] 💀 보안 엔지니어의 시선: 권한 상승 (Privilege Escalation) 취약점
>`vi`나 `vim`이 `sudo` (관리자 권한)로 실행 중일 때, 내부에서 **`:!/bin/sh`** 또는 **`:!/bin/bash`** 를 입력하면, 에디터를 빠져나와 **Root 권한의 쉘(Shell)**이 그대로 열려버린다. 
>따라서 실무 서버에서는 일반 사용자에게 `sudo vi` 권한을 함부로 부여하면 서버가 1초 만에 완벽하게 장악(Takeover)당할 수 있다. (대안: `sudoedit` 사용)

## 🗂️ 리눅스 사용자 및 그룹 관리 핵심 파일

> [!tip] 💡 파일명을 클릭하면 해당 파일의 상세 구조(필드)로 이동합니다.

| 위 치 (경로)                                                       | 의 미 (역할)                                                                              |
| :------------------------------------------------------------- | :------------------------------------------------------------------------------------ |
| [[#1. `/etc/passwd` 구조\|/etc/passwd]]                          | 사용자 계정 정보의 저장 장소 (계정명, UID, GID, 홈 디렉토리, 쉘 정보 등)                                      |
| [[#2. `/etc/shadow` 구조\|/etc/shadow]]                          | 안전하게 보호될 사용자 계정 정보 (암호화된 비밀번호 해시, 만료일 등)                                              |
| [[#3. `/etc/group` 구조\|/etc/group]]                            | 계정이 속한 그룹 ID 및 그룹 목록들에 대한 정보                                                          |
| `/etc/gshadow`                                                 | 그룹의 암호 정보 저장 장소 (거의 사용되지 않음)                                                          |
| `/home`                                                        | 사용자 계정 개별 홈 디렉토리가 생성되는 기본 디렉토리                                                        |
| [[#4. 계정 생성 기본 설정 파일\|/etc/login.defs & /etc/default/useradd]] | 계정 생성 시 참조하는 기본설정 정보<br>(개별 사용자 별 설정은 [[#2. `/etc/shadow` 구조\|/etc/shadow]] 이 우선 적용됨) |
| 계정 관리 명령어                                                      |                                                                                       |

### 1. `/etc/passwd` 구조
> [!example] 데이터 포맷 (콜론 `:` 으로 구분)
> `test : x : 1000 : 1000 : : /home/test : /bin/bash`

| 필드 | 의미 |
| :--- | :--- |
| `test` | 사용자 계정 명 |
| `x` | 사용자 계정에게 부여된 패스워드 (실제 패스워드는 shadow 파일에 저장됨을 의미) |
| `1000` | 숫자로 표현되는 사용자 계정 ID (UID) |
| `1000` | 계정에 대한 숫자로 표현되는 주 그룹 ID (GID) |
| `NULL` | 계정에게 부여되는 임의의 정보 (Comment) |
| `/home/test` | 사용자 계정의 개별 홈 디렉토리 위치 |
| `/bin/bash` | 사용자가 시스템에 로그인 시 사용하는 기본 Shell의 위치 |

---

### 2. `/etc/shadow` 구조
>[!example] 데이터 포맷 (콜론 `:` 으로 구분)
> `test : $6$MFZO03sNMhvGgRih$3ZiWy : 20110 : 0 : 99999 : 7 : : :`

| 필드        | 의미                           |
| :-------- | :--------------------------- |
| `test`    | 사용자 계정 명                     |
| `$6$……$…` | 사용자 계정에게 부여된 암호화된 패스워드 (해시값) |
| `20110`   | 암호 생성일 (1970년 1월 1일 기준 일수)   |
| `0`       | 최소 암호 사용기간                   |
| `99999`   | 최대 암호 사용기간                   |
| `7`       | 암호 만료 전 경고일                  |
| `null`    | 암호 만료 후 유예기간                 |
| `null`    | 계정 만료일                       |

---

### 3. `/etc/group` 구조
> [!example] 데이터 포맷 (콜론 `:` 으로 구분)
> `test : x : 1000 : `

| 필드 | 의미 |
| :--- | :--- |
| `test` | 그룹 명 |
| `x` | 그룹 패스워드 |
| `1000` | 그룹 ID (GID) |
| `null` | 그룹 구성원의 사용자 계정 명 (다수일 경우 콤마로 구분) |

### 1. `useradd` (계정 생성)
> [!example] 💻 실무 예시 명령어
> `sudo useradd -m -d /home/newdev -s /bin/bash -g 1000 -c "New Developer" newdev`

| 옵션   | 설명 (예시 명령어 매핑)                                                           |
| :--- | :----------------------------------------------------------------------- |
| `-m` | **홈 디렉토리 생성** (명령어의 `-m` 부분. 이 옵션이 없으면 폴더가 안 생김)                         |
| `-d` | **홈 디렉토리 위치 지정** (명령어의 `-d /home/newdev` 부분)                             |
| `-s` | **로그인 Shell 설정** (명령어의 `-s /bin/bash` 부분. 보안 계정은 `/usr/sbin/nologin` 사용) |
| `-g` | **기본 그룹의 ID 또는 이름으로 설정** (명령어의 `-g 1000` 부분)                             |
| `-c` | **코멘트, 설명 추가** (명령어의 `-c "New Developer"` 부분. `/etc/passwd`에 기록됨)        |
| `-p` | 비밀번호 설정 (보안상 명령어에 직접 치지 않고 생성 후 `passwd` 명령어로 설정하는 것이 정석)                |
| `-u` | 계정의 UID 수동 설정                                                            |
| `-U` | 계정명과 동일한 이름의 그룹을 자동으로 생성                                                 |
| `-r` | **시스템 계정으로 생성** (UID 1000 미만의 서비스 구동용 계정 생성 시 사용)                        |

---

### 2. `usermod` (계정 수정)
> [!example] 💻 실무 예시 명령어
> `sudo usermod -l superdev -d /home/superdev -m -aG sudo newdev`

| 옵션 | 설명 (예시 명령어 매핑) |
| :--- | :--- |
| `-l` | **계정 명 변경** (명령어의 `-l superdev` 부분. `newdev`를 `superdev`로 바꿈) |
| `-d` | **홈 디렉토리 위치 변경** (명령어의 `-d /home/superdev` 부분) |
| `-m` | **홈 디렉토리 실제 이동** (명령어의 `-m` 부분. `-d`와 함께 써야 기존 파일들이 새 폴더로 이사 감) |
| `-G` | **추가 그룹 설정** (명령어의 `-aG sudo` 부분. `-a`를 안 붙이면 기존 그룹에서 쫓겨나니 주의!) |
| `-p` | 비밀번호 변경 |
| `-u` | 계정 UID 변경 |
| `-g` | 기본 그룹(Primary Group) 변경 |
| `-c` | 코멘트, 설명 변경 |
| `-s` | 로그인 Shell 변경 |

---

### 3. `userdel` (계정 삭제)
> [!example] 💻 실무 예시 명령어
> `sudo userdel -r baduser`

| 옵션 | 설명 (예시 명령어 매핑) |
| :--- | :--- |
| `-r` | **계정 생성 시 정보 모두 삭제** (명령어의 `-r` 부분. 홈 디렉토리와 메일 스풀까지 깔끔하게 소각함. 이거 안 쓰면 찌꺼기 남음) |
| `-f` | **강제 삭제** (사용자가 로그인 중이거나 프로세스가 돌고 있어도 자비 없이 삭제) |

---

### 4. `passwd` (패스워드 관리)
> [!example] 💻 실무 예시 명령어
> `sudo passwd -l hacker` (잠금) / `sudo passwd -u hacker` (해제)

| 옵션 | 설명 (예시 명령어 매핑) |
| :--- | :--- |
| `-l` | **패스워드 잠금 설정** (명령어의 `-l` 부분. Lock의 약자. `/etc/shadow`의 해시값 앞에 `!`를 붙여 로그인을 원천 차단) |
| `-u` | **패스워드 잠금 해제 설정** (명령어의 `-u` 부분. Unlock의 약자) |
| `-d` | **패스워드 삭제** (비밀번호 없이 로그인 가능하게 만듦. 실무에서 쓰면 징계 사유) |

---

### 5. `sudo` & `su` (권한 제어)
> [!example] 💻 실무 예시 명령어
> `sudo cat /etc/shadow` (일회성 권한 빌리기)
> `su - root` (아예 관리자로 영혼 전환)

| 명령어    | 설명                                                                   |
| :----- | :------------------------------------------------------------------- |
| `sudo` | **관리자 권한으로 명령어를 1회 실행** (SuperUser DO. 내 비밀번호를 치고 관리자의 힘을 잠깐 빌림)     |
| `su`   | **계정 변경** (Switch User. `su - 계정명`을 치면 해당 계정의 환경변수까지 완벽하게 흡수하여 로그인함) |
