# LEGECY
## 💀[RCA] 1. 왜 띄어쓰기로 문자를 붙일 수 없는가? (The Delimiter Rule)

> [!danger] 🚨 아키텍트의 팩트 폭격: "컴퓨터에게 띄어쓰기(Space)는 '단어의 끝'을 알리는 구분자(Delimiter)다. 띄어쓰기로 문자를 붙이겠다는 건, 벽돌과 벽돌 사이에 시멘트(+)를 안 바르고 그냥 허공에 띄워놓겠다는 미친 짓이다."

- **V8 엔진의 시선:** 강도님이 `"안녕" "하세요"` 라고 치면, 브라우저 엔진은 이걸 하나의 문장으로 보지 않는다. "어? '안녕'이라는 데이터가 끝났네? 근데 왜 갑자기 뒤에 '하세요'라는 알 수 없는 데이터가 또 튀어나와?" 하고 뇌 정지가 오며 **`Uncaught SyntaxError`**를 뱉고 장렬하게 뻗어버린다.
- **`+` 연산자의 진짜 의미:** `+`는 숫자일 때는 덧셈(ALU 연산)을 하지만, 문자일 때는 **"앞의 메모리 주소와 뒤의 메모리 주소를 하나로 이어 붙여라(Concatenation)"**라는 명확한 메모리 할당 명령어다.

## 🚀 [Architecture] 2. `+`가 귀찮은 자들을 위한 구원: 템플릿 리터럴 (ES6)

강도님처럼 `+`와 따옴표(`""`)를 닫고 여는 것에 빡친 전 세계 개발자들이 2015년에 새로운 문법을 창조했다. **이걸 쓰면 강도님이 원하는 '띄어쓰기'가 10000% 가능해진다.**

- **레거시 (강도님이 빡친 방식):** 
  `let text = "내 이름은 " + name + " 이고, 나이는 " + age + " 살이다.";`
- **모던 JS (백틱 ` ` 의 마법):** 키보드 물결표(`~`) 자리에 있는 **백틱(Backtick)**으로 감싸면 끝난다.
  `let text = \`내 이름은 ${name} 이고, 나이는 ${age} 살이다.\`;`
  👉 **팩트 폭격:** 백틱 안에서는 띄어쓰기, 엔터(줄바꿈)가 전부 그대로 인식된다. 변수는 `${ }` 안에 꽂아 넣기만 하면(Injection) 끝이다. (이게 현대 JS의 절대 표준이다.)

## 🛡️ [Security] 3. DevSecOps가 반드시 알아야 할 JS 3대 생존 법칙

자바스크립트는 10일 만에 대충 만들어진 언어라 태생적으로 버그가 미친 듯이 많다. 아래 3가지를 어기면 서버가 털리거나 화면이 박살 난다.

### 🚨 법칙 1: `==` 은 쓰레기다. 무조건 `===` 을 써라. (Type Coercion)
- **`==` (느슨한 비교):** `1 == "1"` 을 `true`라고 우기는 미친 연산자다. (숫자와 문자를 지 맘대로 변환해서 비교함)
- **`===` (엄격한 비교):** `1 === "1"` 은 `false`다. 값과 '데이터 타입(Type)'까지 완벽하게 일치해야만 인정한다.
- **보안 엔지니어의 시선:** 해커가 권한 체크 로직을 우회(Auth Bypass)할 때 `==`의 허술함을 찌른다. 실무에서 `==`를 쓰면 사수한테 뺨을 맞는다.

### 🚨 법칙 2: `var` 는 죽었다. `let` 과 `const` 만 써라. (Scope Hoisting)
- **`var` (레거시):** 변수를 선언하면 전역(Global)으로 오염되어, 엉뚱한 곳에서 값이 덮어씌워지는 대참사(Memory Leak)가 발생한다.
- **`let` (변수) / `const` (상수):** 자신이 선언된 중괄호 `{ }` 블록 안에서만 생존하는 안전한 샌드박스(Sandbox) 변수다.

### 🚨 법칙 3: 세미콜론(`;`)을 생략하지 마라. (ASI의 배신)
- JS는 줄 끝에 세미콜론을 안 써도 브라우저가 알아서 붙여주는 기능(ASI)이 있다.
- **팩트 폭격:** 이 멍청한 자동 완성 기능을 믿었다가, 엉뚱한 줄에서 코드가 끊겨서 프로덕션(실서버) 화면이 백지화되는 사고가 매년 일어난다. 문장이 끝나면 무조건 **`;`** 을 찍는 습관을 들여라.

/* 	
 	true && true  -> true
	
	true && false -> false
	false && true -> false
	
	false && false -> false
	------------------------------
	true || true  -> true
	
	true || false -> true
	false || true -> true
	
	false && false -> false 
*/
	
	var x = 5, y = 7;
	document.write("(x < 10 && y > 10) : " + (x < 10 && y > 10) + "<br>");
	document.write("(x < 10 || y > 10) : " + (x < 10 || y > 10) + "<br>");
	document.write("!(x < 10 && y > 10) : " + !(x < 10 && y > 10) + "<br>");

	// true or false ? true일때 실행할 문장 : false일 때 실행할 문장;
		
	result = (x > y) ? x : y; // 조건 연산
	document.write("큰 값 : " + result + "<br>");
	result = (x < y) ? x - y : y - x; // 조건 연산
	document.write("큰 값 - 작은 값 : " + result + "<br>");
	</script>
